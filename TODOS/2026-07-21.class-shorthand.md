# `:class.<name>` — the single-flag shorthand

**Status:** proposal. Builds on [`2026-07-15.class-directive.md`](2026-07-15.class-directive.md).

## The problem

The object form of `:class` earns its keep for a set of flags, but the
overwhelmingly common case is *one* class gated by *one* condition:

```html
<div class="drop" :class="{ active: dropping }"></div>
```

The `{ … }` and the quoting-in-a-quote are ceremony around a single boolean.
Every framework that has the object form grows a per-class shorthand next to it
(Vue's `:class.active`, Angular's `[class.active]`) for exactly this reason.

## The proposal

`:class.<name>="expr"` toggles the single class `<name>` on when `expr` is
truthy — an alias for `:class="{ <name>: expr }"`:

```html
<div class="drop" :class.active="dropping"></div>
```

- **`<name>`** is everything after `:class.`. The HTML parser lowercases
  attribute names, so the class token is lowercased with it: kebab-case names
  survive (`:class.is-active`), camelCase does not (`:class.isActive` →
  `isactive`). Same limitation as `@event` modifiers and `:model.<name>`, and
  the same fix (write the name kebab-case).
- **truthiness** is the object-form value rule: any truthy value adds the
  class, anything falsy contributes nothing — so `:class.busy="pending && !done"`
  reads naturally.
- **it coexists with `:class`** and with other `:class.<name>` on the same
  element; the sets union. `class="drop" :class="theme" :class.active="dropping"`
  is fine.

It reuses the `:class` effect and merge machinery wholesale, so the invariant is
unchanged: only classes the binding added last run are ever removed, and the
static `class` list is never touched.

## What it costs

- `:class.*` must be skipped by the static-attr loop (or it's written as a
  literal attribute) and by nested-component prop collection (or `:class.active`
  becomes a prop `classActive`). Both already gate on `CONTROL_ATTRS.has`; they
  move to an `isControlAttr(attr)` helper that also matches the `:class.` prefix
  — the exact-match set can't hold an open-ended name.
- The class effect gains a `toggles` list (the `:class.*` attrs, name +
  expression) and now runs when *either* a bare `:class` or a toggle is present.
  ~4 lines.

## Rejected

- **A separate effect per toggle.** The "remove only what I added last run"
  bookkeeping and the single `classList` pass live in one effect for `:class`;
  splitting toggles out would duplicate the bound-set tracking and let two
  effects race on the same `classList`. Fold them into the one effect.
- **Multiple names in one attribute (`:class.a.b`).** A class token can't hold a
  dot, and the object/array forms already express multi-class. One attribute,
  one class.
- **Working on a component tag.** `:class` is already ignored there (jq79
  components have no single root to fall through to); the shorthand follows,
  ignored alongside it.

## Open

- camelCase names are unreachable (the parser lowercases them) — the shared
  limitation across every dotted directive, not worth a per-directive escape.

## 2026-07-21 — implemented

As proposed. An `isControlAttr` helper (`CONTROL_ATTRS.has(attr) ||
attr.startsWith(":class.")`) replaced the two bare `CONTROL_ATTRS.has` checks in
the static-attr loop and the nested-component prop collector. The class effect
collects `:class.*` into `[name, expr]` pairs, runs when a bare `:class` *or* a
toggle exists, seeds `next` from the bare expression (empty when absent), then
pushes each toggle's `classNames(name)` whose expression is truthy — routing the
name through `classNames` so an empty `:class.` can't reach `classList.add`
(which throws on `""`). Removal/merge are untouched.

Tests cover a single toggle flipping on and off over the static class, a toggle
unioning with a bare `:class`, two toggles on one element, kebab names, the
static class surviving a toggle that names then drops it, and a toggle on a
`:each` item. `docs/template-syntax.md` documents the shorthand under `:class`.
